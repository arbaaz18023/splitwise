# Backend Authentication API Contract

This document describes the API contract for authenticating mobile users using the **Continue with Google** sign-in option.

---

## Endpoint Specification

- **HTTP Method**: `POST`
- **Path**: `/api/auth/google`
- **Content-Type**: `application/json`
- **Accept**: `application/json`

---

## Request Payload

The client sends the Google Identity Provider ID token (`idToken`) obtained from the Google Sign-in SDK on Android.

```json
{
  "idToken": "string"
}
```

### Fields

| Field Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `idToken` | String | Yes | The cryptographically signed OIDC ID Token (JWT) issued by Google. |

---

## Success Response

- **Status Code**: `200 OK`

The backend verifies the ID token with Google's servers. If valid, it returns a session/authorization token and the verified user profile information.

```json
{
  "token": "string",
  "refreshToken": "string",
  "user": {
    "id": "string",
    "name": "string",
    "email": "string",
    "avatarUrl": "string"
  }
}
```

### Fields

| Field Name | Type | Description |
| :--- | :--- | :--- |
| `token` | String | The session token or JWT generated by the backend to authenticate subsequent API calls (should be passed in the `Authorization: Bearer <token>` header). |
| `refreshToken` | String | The long-lived refresh token used to request new session tokens without requiring user re-authentication. |
| `user` | Object | The user profile object. |
| `user.id` | String | The unique user ID on the backend database. |
| `user.name` | String | The user's full display name (retrieved from Google token). |
| `user.email` | String | The user's verified email address (retrieved from Google token; can be null if not shared). |
| `user.avatarUrl` | String | URL to the user's avatar image (can be null/omitted if unavailable). |

---

## Error Responses

### 1. Verification Failed (e.g., Invalid or Expired Token)
- **Status Code**: `401 Unauthorized`

```json
{
  "error": "Unauthorized",
  "message": "Invalid google credential. Backend verification failed."
}
```

### 2. Bad Request (e.g., Missing Parameter)
- **Status Code**: `400 Bad Request`

```json
{
  "error": "Bad Request",
  "message": "Required request body parameter 'idToken' is missing."
}
```

---

## Error Responses

### 1. Missing or Invalid Token
- **Status Code**: `401 Unauthorized`

```json
{
  "error": "Unauthorized",
  "message": "Full authentication is required to access this resource."
}
```

---

# Token Refresh API Contract

This section describes the API contract to exchange a valid refresh token for a fresh short-lived session token.

## Endpoint Specification

- **HTTP Method**: `POST`
- **Path**: `/api/auth/refresh`
- **Content-Type**: `application/json`
- **Accept**: `application/json`

---

## Request Payload

```json
{
  "refreshToken": "string"
}
```

### Fields

| Field Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `refreshToken` | String | Yes | The active refresh token previously issued to the client during sign-in or token refresh. |

---

## Success Response

- **Status Code**: `200 OK`

```json
{
  "token": "string",
  "refreshToken": "string"
}
```

### Fields

| Field Name | Type | Description |
| :--- | :--- | :--- |
| `token` | String | The new session/access token to authenticate future requests. |
| `refreshToken` | String | A new refresh token (if rotating refresh tokens is enabled by backend; can be null/omitted if reusing original). |

---

## Error Responses

### 1. Expired or Invalid Refresh Token
- **Status Code**: `401 Unauthorized`

```json
{
  "error": "Unauthorized",
  "message": "Invalid or expired refresh token. User must re-authenticate."
}
```

### 2. Bad Request
- **Status Code**: `400 Bad Request`

```json
{
  "error": "Bad Request",
  "message": "Required request body parameter 'refreshToken' is missing."
}
```

# Logout API Contract

This section describes the API contract for invalidating the active user session token upon logout.

## Endpoint Specification

- **HTTP Method**: `POST`
- **Path**: `/api/auth/logout`
- **Headers**:
  - `Authorization: Bearer <token>` (Required)
- **Accept**: `application/json`

---

## Request Payload

No request body is required.

---

## Success Response

- **Status Code**: `200 OK`

```json
{
  "message": "Successfully logged out and session token invalidated."
}
```

---

## Error Responses

### 1. Missing or Invalid Token
- **Status Code**: `401 Unauthorized`

```json
{
  "error": "Unauthorized",
  "message": "Full authentication is required to access this resource."
}
